Token-saving skills for LLM agents.

Cut output and context tokens without choking the model. No caveman persona. Measured, not claimed.

Abstract

Four modular skills compress agent verbosity, trim context before overflow, strip preamble, and enforce a hard token budget — without altering code or numbers. In a 10-turn synthetic harness the median output dropped 53.8% with zero correctness regression on a reasoning model. The benchmark is reproducible.

§02 — Capabilities

Six skills, one goal

  • paleo

    Aggressive verbosity compression. Drop filler, keep code and numbers exact.

  • paleo-trim-context

    Proactively trim context before it overflows the window.

  • paleo-budget

    Hard token ceiling — auto-summarize when over budget.

  • paleo-converse

    Condense old chat turns and merge duplicate messages.

  • paleo-summary

    Tight intisari of bulky tool output, logs, and diffs.

  • paleo-json

    Minify and compact structured/JSON output, still parseable.

§05 — Positioning

Paleo, Ponytail & Caveman

  • paleo

    Output + context + conversation + hard budget. 53.8% measured. Replaces Caveman, complements Ponytail.

  • ponytail

    Conversation-only compression. A different layer — stack it under paleo if you want both.

  • caveman

    ~30–40% on output, persona-based. Paleo covers the same surface with open benchmarks.
